# PRD: 소상공인 글로벌 마케팅 자동화 플랫폼

## 목표
소상공인이 해외 고객 대상 마케팅 업무를 직접 하지 않아도 되도록, 구조화된 입력만으로 콘텐츠 생성부터 리뷰 대응과 성과 리포트까지 자동화한다.

## 문제 정의
- 소상공인은 해외 고객 대상 채널 운영을 직접 수행할 시간과 전문성이 부족하다.
- 다국어 콘텐츠와 리뷰 대응은 품질 편차와 운영 부담이 크다.
- 지원기관과 운영 담당자는 점포별 진행 현황과 성과를 일관되게 추적하기 어렵다.
- 챗봇형 입력은 누락과 오입력이 많아 실제 운영 자동화로 연결되기 어렵다.

## 사용자
- 해외 고객을 유치하고 싶은 소상공인 점주
- 지자체, 관광기관, 상권활성화 조직 담당자
- 소상공인 지원사업 운영 담당자
- 내부 운영 매니저 또는 마케팅 대행 파트너

## 사용자 가치
- 점주는 복잡한 마케팅 도구 대신 짧고 명확한 입력만 제공하면 된다.
- 운영자는 승인, 게시, 리뷰, 리포트 이력을 한 곳에서 관리할 수 있다.
- 지원기관은 사업 단위 성과와 점포별 활동 현황을 감사 가능하게 확인할 수 있다.

## 핵심 기능
1. 점포 사진과 기본정보를 기반으로 브랜드 프로필 자동 생성
2. 나라, 플랫폼, 목적, 홈페이지, 이미지 등을 입력받는 콘텐츠 생성 모달 제공
3. 국가/플랫폼별 마케팅 콘텐츠 자동 생성 및 승인 기반 발행
4. 업로드한 입력 이미지를 기반으로 발행 전 이미지 보정/변형 초안 생성
5. 외국어 리뷰 분류, 답글 초안 생성, 민감 리뷰 escalation
6. Google/Instagram/기타 채널용 소개문, FAQ, SEO/AEO 문안 생성
7. 점포/사업 단위 월간 성과 리포트 자동 생성
8. 관리자용 대시보드에서 점포별 활동, 승인, 게시, 성과 추적

## 주요 사용자 시나리오

### 1. 점주가 콘텐츠를 요청한다
- 점주는 콘텐츠 생성 모달에서 국가, 플랫폼, 목적, 강조 문구, 홈페이지 URL, 이미지를 입력한다.
- 시스템은 점포 프로필과 기존 승인 콘텐츠를 참고해 초안을 생성한다.
- 점주가 이미지를 올린 경우, 발행 시점에 입력 이미지를 기반으로 한 보정/변형 이미지를 추가 생성할 수 있다.
- 결과는 바로 게시되지 않고 승인 대기 상태로 저장된다.

### 2. 리뷰 대응을 처리한다
- 외부 플랫폼에서 리뷰가 수집된다.
- 시스템은 언어, 감성, 민감도를 분류한다.
- 일반 리뷰는 답글 초안을 만들고, 민감 리뷰는 운영자 검토 대상으로 escalation 한다.

### 3. 운영자가 성과를 확인한다
- 운영자는 대시보드에서 점포별 게시 수, 승인 대기 건수, 리뷰 처리 상태, 월간 성과를 확인한다.
- 필요 시 점포나 사업 단위로 리포트를 생성하거나 다운로드한다.

## 성공 지표
- 콘텐츠 요청부터 초안 생성까지 평균 처리 시간이 수동 작업 대비 유의미하게 감소한다.
- 승인 후 게시 성공률이 일정 기준 이상 유지된다.
- 리뷰 응답 초안 생성률과 운영자 승인 전환율이 추적 가능하다.
- 관리자 화면에서 점포별 활동 이력과 감사 로그가 누락 없이 조회된다.

## MVP 범위
- 단일 또는 소수 국가를 대상으로 한 콘텐츠 생성
- 제한된 플랫폼 연동
- 승인 기반 발행
- 입력 이미지 기반 보정/변형 이미지 생성
- 리뷰 분류 및 답글 초안
- 월간 성과 리포트
- 관리자 활동 대시보드

## MVP 제외 사항
- 자유 대화형 챗봇 인터페이스
- 자동 이미지 생성 및 고급 디자인 편집
- 완전 자동 광고 집행 및 예산 최적화
- 실시간 다국어 음성 상담 에이전트
- 오프라인 POS/결제 시스템 완전 통합
- 모든 국가와 플랫폼의 동시 지원

## 제품 원칙
- 사용자 입력은 자연어보다 구조화된 폼을 우선한다.
- AI는 사용자 인터페이스가 아니라 내부 실행 엔진으로 동작한다.
- 생성 결과는 승인과 감사 가능성을 전제로 설계한다.
- 운영자는 결과물뿐 아니라 과정과 상태를 추적할 수 있어야 한다.
- 이미지 생성은 텍스트만으로 새로 만드는 방식보다, 점주가 올린 입력 이미지를 기반으로 한 변형/보정 흐름을 우선한다.
- 점주/관리자 UI는 `frontend/` 아래 Next.js App Router로 구현하고, 백엔드 `backend/app/`와 분리한다.
- `frontend/`와 `backend/`를 나누는 이유는 Python 백엔드 패키지와 프론트 라우트 루트의 역할을 분리하고, Docker Compose에서 서비스별 빌드와 실행을 명확하게 하기 위해서다.

## 비기능 요구사항
- 모든 외부 발행 가능 콘텐츠와 민감 리뷰는 감사 로그를 남긴다.
- 승인 이력, 작업 상태, 결과물 버전은 DB에 기록한다.
- 실패한 작업은 재시도 또는 재개 가능해야 한다.
- 시스템은 B2B/B2G 환경에서 설명 가능해야 한다.

## 디자인 방향
- 점주 화면은 입력 단계가 짧고 명확한 폼 중심 UX를 사용한다.
- 관리자 화면은 정보 밀도가 높되 복잡하지 않은 실무형 B2B SaaS UI를 지향한다.
- 전체 톤은 깔끔하고 신뢰감 있는 업무 도구 느낌을 유지한다.
- 색상은 무채색 기반에 블루/그린 포인트를 사용한다.
- UI 구현은 Next.js App Router 기준이며, 모달과 폼 상태는 클라이언트 컴포넌트에서 관리한다.
